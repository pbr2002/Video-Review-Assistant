# Video-Review-Assistant
# 🎬 视频审核助手（Tkinter GUI）

一个基于 **Python + Tkinter + openpyxl** 的轻量级人工视频审核工具，支持从 Excel 批量加载视频链接，并快速进行审核标记。

---

## ✨ 项目特点

* 🖥️ 纯 Tkinter 实现，无需额外 GUI 框架（跨平台：macOS / Windows）
* 📊 支持 Excel 批量审核（自动识别列）
* 🔗 自动打开视频链接（浏览器）
* ⚡ 高效操作流：

  * 点击“审核通过”即可完成
  * **选择“不通过原因”即可自动判定并跳转下一条（核心优化）**
* 💾 每条审核后自动保存（支持断点续跑）
* ⏭️ 自动跳过已审核记录
* 🧠 智能解析 Excel 超链接（含 HYPERLINK）

---

## 🚀 功能说明

### 核心流程

1. 选择 Excel 文件
2. 加载数据
3. 自动定位未审核记录
4. 打开视频链接
5. 执行审核：

| 操作      | 行为             |
| ------- | -------------- |
| ✅ 审核通过  | 点击按钮           |
| ❌ 审核不通过 | **选择原因（自动提交）** |
| ⏭️ 跳过   | 手动跳过           |

---

### ⭐ 核心亮点（高效审核）

> 选择“不通过原因” = 自动完成审核 + 保存 + 跳下一条

👉 无需再点击“审核不通过”按钮
👉 审核效率提升明显（适合大批量处理）

---

## 📂 Excel 格式要求

支持自动识别以下列（无需严格匹配名称）：

| 字段   | 示例          |
| ---- | ----------- |
| 视频ID | 视频ID        |
| 视频链接 | 视频链接 / 投稿链接 |
| 平台   | 平台          |
| 审核结果 | 审核结果        |
| 原因   | 原因          |

### 必填列：

* 视频链接
* 审核结果
* 原因

## 🧠 设计细节

### ✔ 自动保存机制

每次审核后立即写回 Excel，避免数据丢失

### ✔ 防误触机制

* 加载数据时不会触发自动审核
* 仅用户选择原因时才触发

### ✔ UI 优化

* 当前行号高亮显示（顶部 + 标题）
* 状态提示实时更新
* 自动刷新后再打开浏览器（避免 UI 卡顿）

---

## ⚠️ 注意事项

* 程序会**直接覆盖原 Excel 文件**
* 请确保 Excel 文件未被占用（关闭 Excel/WPS）
* 不通过原因必须从列表中选择
